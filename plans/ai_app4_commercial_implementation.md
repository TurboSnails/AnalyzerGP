# ai_app4 商业级别实施计划：三轨融合检索

> 本地 RAG + 三方金融 API + 网络搜索 的融合架构

---

## 二、新增组件清单

### 2.1 DataSource 抽象层（rag_framework 扩展）

文件：`rag_framework/rag_framework/datasource/base.py`

职责：统一所有数据源（本地检索、金融 API、网络搜索）的接口，返回标准化的 `RetrievedDoc`。

关键设计：
- `fetch()` 必须处理 API 限流、超时、降级，绝不抛异常中断主流程
- 返回 `list[RetrievedDoc]`，与本地 RAG 结果格式完全一致
- `ttl_seconds` 控制缓存策略

### 2.2 金融 API 客户端（Track B）

| 数据源 | 用途 | 接入方式 | 商业成本 |
|--------|------|---------|---------|
| **Yahoo Finance** | 股价、市值、PE、财报日历 | `yfinance` 库 | 免费（有限流） |
| **Alpha Vantage** | 历史价格、技术指标 | REST API | 免费 25 次/天，付费 $49/月起 |
| **FRED** | CPI、非农、利率、GDP | REST API（免费 Key） | 免费 |

文件：
- `ai_app4/service/datasources/yahoo_finance.py`
- `ai_app4/service/datasources/fred_api.py`
- `ai_app4/service/datasources/alpha_vantage.py`

### 2.3 网络搜索客户端（Track C）

| 服务商 | 特点 | 成本 | 适用场景 |
|--------|------|------|---------|
| **Tavily** | 专为 AI RAG 优化，返回清洗后的文本摘要 | $0.025/次 | 通用搜索（推荐首选） |
| **SerpAPI** | Google 搜索结果结构化 | $0.005/次 | 多源对比 |
| **Bing Search API** | 微软官方，中文可用性好 | $7/1000次 | 中文搜索优先 |

文件：
- `ai_app4/service/datasources/tavily_search.py`

### 2.4 三轨融合检索器

文件：`ai_app4/service/retrieval/three_track_retriever.py`

职责：
- 编排本地 HybridRetriever + 多个 DataSource
- 按权重融合结果
- 来源标注
- 成本控制：分级触发、超时降级

### 2.5 缓存层

文件：`ai_app4/service/cache.py`

职责：
- 结构化金融数据（股价、PE 等）缓存 5 分钟
- 内存 dict + 可选 Redis
- 失败回退到本地 RAG

## 三、成本控制策略

| 策略 | 实现方式 | 预估节省 |
|------|---------|---------|
| **分级触发** | classify 节点判断：历史知识类问题只用 Track A（免费），实时问题才启用 Track B/C | 60-80% |
| **实体缓存** | 股价/PE 等结构化数据缓存 5 分钟，同一会话内重复查询命中缓存 | 30-50% |
| **搜索配额** | 单用户每日搜索上限 10 次，超限提示"搜索额度已用完" | 防止恶意刷量 |
| **批处理聚合** | 涉及多只股票时，一次性批量查询而非逐只串行 | 减少 API 调用 50% |
| **降级兜底** | 所有三方 API 失败时静默返回空，LLM 基于本地知识回答并提示"数据可能不是最新的" | 零中断 |

## 四、LangChain 定位（最终结论）

| 场景 | 建议 |
|------|------|
| LLM 调用 | 保持 openai SDK（稳定性第一） |
| 检索管道 | 自研三轨融合（核心竞争壁垒） |
| Prompt 管理 | 引入 `langchain-core` PromptTemplate（标准化） |
| 三方数据接入 | 可选 LangChain Community Loaders 加速开发，但数据获取后立即转为 `RetrievedDoc` |
| 网络搜索 | 直接用 Tavily/SerpAPI SDK（LangChain 只是薄封装） |

## 六、关键风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Yahoo Finance 限流/封 IP | 股价数据获取失败 | 降级到 Alpha Vantage，或提示"实时数据暂不可用" |
| Tavily 账单爆炸 | 运营成本失控 | 单用户日限额 + 按租户配额 + 超量自动关闭搜索 |
| API 响应慢（>5s） | 用户体验差 | 所有三方 API 设置 3 秒超时，超时后只返回本地 RAG 结果 |
| 数据幻觉（LLM 编造数据） | 金融场景致命 | 来源强制标注 + prompt 明确禁止编造数字 |
| 合规风险 | 投资建议无免责声明 | 系统 prompt 固定追加免责声明 |
