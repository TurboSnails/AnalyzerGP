# investment_analyzer - 投资分析工作流设计文档

> 版本: 2.0 | 最后更新: 2026-05-08

---

## 1. 系统定位

investment_analyzer 是一个 **逻辑驱动型投资分析框架** 的自动化实现。输入股票代码，系统自动完成从宏观周期定位到个股深度分析的完整流程，输出结构化的 Markdown 分析报告。

**核心理念**：宏观定轨 → ETF 轮动 → Alpha 个股挖掘，三层框架层层递进。

## 4. 核心分析器设计

### 4.1 跌幅量化 (drop_checker.py)

**目标**: 验证股票是否处于「超跌」状态，排除高位接盘风险。

| 检查项 | 阈值 | 权重 |
|--------|------|------|
| 相对历史高点跌幅 | >= 50% | +1 |
| PE 历史分位数 | <= 20% | +1 |
| PB 历史分位数 | <= 20% | +1 |

**通过标准**: score >= 2（至少2项通过）

### 4.2 真反转验证 (reversal_checker.py)

**目标**: 区分「真反转」与「价值陷阱」。

| 维度 | 检查内容 | 数据来源 |
|------|----------|----------|
| 毛利率趋势 | 连续2期改善 | 财务指标表 |
| 经营现金流 | 为正且增速>=0 | 现金流量表 |
| 库存周期 | 周转天数下降 | 资产负债表 |
| 管理层行为 | 增持>减持 | 内部人交易 |
| 行业竞争格局 | 需 AI 辅助 | — |
| 负债结构 | 资产负债率 < 60% | 资产负债表 |
| 核心业务健康度 | 营收增速+毛利率双升 | 利润表 |

**通过标准**: score >= 4/7

### 4.3 护城河验证 (moat_checker.py)

**目标**: 六维度评估企业竞争壁垒。

| 维度 | 检查内容 | 类型 |
|------|----------|------|
| 定价权 | 毛利率>=40% 且稳定 | 定量 |
| 成本优势 | 三费占比下降 | 定量 |
| 规模效应 | 营收增长+毛利同步改善 | 定量 |
| 无形资产 | 研发占比/品牌溢价 | 定量+定性 |
| 转换成本 | 客户迁移难度 | 定性（AI辅助） |
| 网络效应 | 用户增长驱动营收加速 | 定性（AI辅助） |

**通过标准**: score >= 3/6

### 4.4 基本面双轨筛选 (fundamental_screener.py)

**目标**: 将标的归类为 A类（价值/红利）或 B类（第二曲线成长）。

#### A类 - 价值/红利型

| 指标 | 阈值 | 说明 |
|------|------|------|
| ROE | > 15% | 资本回报能力 |
| 资产负债率 | < 60% | 财务安全 |
| 股息率 | > 2% | 股东回报 |
| 经营现金流增速 | > 8% | 造血能力 |
| 扣非净利润增速 | > 10% | 真实盈利增长 |
| 毛利率 | > 20% | 定价能力 |

#### B类 - 第二曲线成长型

| 指标 | 阈值 | 说明 |
|------|------|------|
| Rule of 40 | >= 40 | 营收增速% + 净利润率% |
| 毛利率 | > 40%（科技）/ >30%（消费） | 产品竞争力 |
| 营收加速度 | 连续3季度加速 | 增长动能 |
| 经营现金流改善 | 由负转正或增速>20% | 盈利质量 |

**优先逻辑**: B类优先 → 通过则推荐 B类；否则判断 A类。

### 4.5 宏观定轨 (macro_checker.py)

**目标**: 判断当前市场处于何种周期阶段。

| 信号 | 计算方式 | 判断标准 |
|------|----------|----------|
| MA200 | 沪深300 vs 200日均线 | 当前<MA200 = below |
| PE分位 | 当前PE在历史分布中的位置 | <30%=低估, >70%=高估 |
| 国债收益率 | 60日变化趋势 | >10bp=上行, <-10bp=下行 |

**状态判定表**:

| MA200 | PE分位 | 收益率 | 状态 | 策略 |
|-------|--------|--------|------|------|
| below | low | — | bear_bottom | 定投科技/宽基ETF |
| below | — | rising | rate_hike_bear | 转货币基金 |
| below | — | falling/flat | recession_bear | 转红利ETF |
| 其他 | — | — | ambiguous | 维持现状 |

### 4.6 仓位管理 (position_sizer.py)

**核心规则**:

| 参数 | 值 | 说明 |
|------|-----|------|
| 单标的上限 | 10% | 硬性上限，含浮盈后需再平衡 |
| Alpha层总上限（正常） | 25% | 非熊市时的组合上限 |
| Alpha层总上限（熊市） | 12% | rate_hike_bear / recession_bear |

**确信度 → 仓位档位**:

```
confidence >= 80% → 10% (高确信)
confidence >= 60% →  7% (中确信)
confidence >= 40% →  5% (低确信)
confidence <  40% →  0% (观望)
```

**退出触发器**:
- 营收增速连续2季度减速
- 毛利率持续下滑（连续2季度）
- 核心研发/业务负责人离职
- Rule of 40 跌破 30
- 逻辑假设被证伪

**时间止损**:
- 18个月逻辑成立但市场无反应 → 重新评估
- 24个月无效率 → 减仓至 1/2

## 5. AI Agent 架构

### 5.1 Agent 设计

所有 Agent 继承自 `BaseAgent`，通过 `litellm` 统一调用 LLM（支持 DeepSeek/Claude/GPT/千问）。

```
BaseAgent
    ├── call_llm(prompt, context)    # 统一 LLM 调用接口
    │
    ├── FirstPrinciplesAgent         # 第一性原理分析
    │      └── analyze(company, data_context)
    │
    ├── BullAgent                    # 多头分析师
    │      └── analyze(company, data_context)
    │
    ├── BearAgent                    # 空头分析师
    │      └── analyze(company, bull_case, data_context)
    │
    └── JudgeAgent                   # 投资决策裁判
           └── decide(company, bull_case, bear_case, ...)
```

### 5.2 辩论流程

```
Phase 3: 第一性原理分析
    │
    ▼
Phase 4: 多空辩论
    ├── 多头分析师 (BullAgent) 基于数据提出投资理由
    │
    ├── 空头分析师 (BearAgent) 针对多头观点提出反驳
    │      └─ 输入包含 bull_case，确保反对有的放矢
    │
    └── 裁判综合评估 (JudgeAgent)
           └─ 输入: 宏观环境 + 量化数据 + 仓位建议 + 多头观点 + 空头观点
           └─ 输出: 综合评分 / 投资结论 / 仓位建议 / 退出条件 / 跟踪指标
```

### 5.3 JudgeAgent 决策框架

JudgeAgent 的 system_prompt 编码了完整的投资纪律：

- **仓位原则**: 单标的 10% 上限，Alpha层 25%（熊市 12%）
- **退出原则**: 逻辑失效立即清仓，不以价格为主要退出信号
- **框架优先级**: 宏观状态优先于个股决策

## 6. 数据层设计

### 6.1 市场路由器 (market_router.py)

自动识别输入代码所属市场：

| 输入格式 | 识别规则 | 输出 |
|----------|----------|------|
| 600519, sh600519 | 6开头=上证, 0/3开头=深证 | A股 |
| 00700, 00700.HK | 4-5位数字 + .HK 后缀 | 港股 |
| AAPL, TSLA | 1-5位大写字母 | 美股 |
| 贵州茅台 | 中文名称 → 通过 akshare 搜索 | A股 |

### 6.2 数据源映射

| 市场 | 数据源 | 数据类型 |
|------|--------|----------|
| A股 | akshare | 行情/财务/估值/宏观/内部人 |
| 港股 | yfinance | 行情/财务/内部人 |
| 美股 | yfinance | 行情/财务/内部人 |

### 6.3 缓存机制 (cache.py)

- TTL: 4小时
- 缓存目录: `output/.cache/`
- 装饰器: `@cached(cache_key)` 自动管理缓存读写

## 8. 监控工具 (monitor.py)

独立的观察名单管理工具：

```bash
python monitor.py              # 检查所有观察标的
python monitor.py --add 600519 # 添加到观察名单
python monitor.py --list       # 显示观察名单
```

功能：
- 定期检测观察名单中各标的的超跌信号
- 新触发超跌信号时生成 alert
- 持久化存储: `output/watchlist.json` + `output/monitor_log.jsonl`

## 9. CLI 用法

```bash
# 分析单只A股
python main.py 600519

# 分析美股
python main.py AAPL

# 分析港股
python main.py 00700.HK

# 快速分析（仅量化，不调用AI）
python main.py 600519 --depth quick

# 跳过AI分析
python main.py 600519 --no-ai

# 生成图表
python main.py 600519 --charts

# 附加回测
python main.py 600519 --backtest

# 批量分析
python main.py --batch 600519,AAPL,00700.HK
python main.py --batch-file stocks.txt
```

## 10. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 分析框架 | 逻辑驱动型三层框架 | 自上而下：宏观 → 行业 → 个股 |
| 数据源 | akshare + yfinance | 免费、覆盖 A股/港股/美股 |
| LLM 接口 | litellm | 统一接口，支持多模型切换 |
| 缓存 | 文件系统 + TTL | 简单可靠，避免重复请求 |
| 报告格式 | Markdown | 易读、易分享、可版本控制 |
| 量化+AI 分工 | 量化筛选前置，AI 定性补充 | 避免 AI 幻觉导致量化错误 |
| 多空辩论 | Bull vs Bear + Judge | 强制检验投资逻辑的对立面 |
| 仓位规则 | 固定上限 + 确信度档位 | 不使用凯利公式（参数不可量化） |
