# 📑 需求定义：全球资产与宏观经济多步推演智能助理 (Wealth AI Agent v4.0)

## 1. 业务场景与价值

作为一名关注全球资产配置（美股科技股、A股/港股、行业龙头如比亚迪）的投资者，市场信息极其碎片化：

* **静态宏观数据**：CPI、PPI、VIX 走势，美联储议息会议纪要（纯英文长文本）。
* **动态企业财报**：财报 PDF 中的毛利率变化、营收指引（多模态或复杂长文本）。
* **交易策略逻辑**：网格交易法、做 T 降低成本算法、凯利公式仓位控制。

**核心痛点**：普通 RAG 只能回答“NVIDIA 财报营收是多少？”，无法回答“**结合美联储最新 PPI 走势和 NVIDIA 最新的营收指引，我的网格交易策略需要做何种仓位调整？**”（即多跳 Multi-hop 推理与数学计算的结合）。

---

## 2. 核心功能需求（功能矩阵）

### 2.1 多领域数据录入与隔离（复用 v3.4/v3.5 统一 Collection 架构）

* 将下载的**券商宏观研报、美联储会议纪要、科技股财报 PDF/Markdown** 录入统一的 `knowledge_base`。
* 扩展 `domain` 元数据字段：`domain: "macro_econ"`（宏观）、`domain: "corp_earnings"`（财报）。保持现有的 `android` 和 `msmarco` 作为对照组。

### 2.2 跨语言与跨领域智能路由（升级 `rewrite_router_rules`）

* 用户输入中文，但涉及美联储时，系统应自动分流：
* **本地 Qwen（L2改写）** 将 Query 拆解、翻译并派发出并行的子查询（如：`NVIDIA earnings guidance` + `Fed PPI inflation impact`）。
* 并发检索 `macro_econ`（英文源）和 `corp_earnings`（中英文源），并进行跨领域融合（`domain: "all"`）。



### 2.3 基于 LangGraph 的“反思与自旋纠错”控制流（核心重构点）

* 系统不再是一条链走到底，而是通过状态机控制：
1. **分类与改写** -> 2. **并发检索** -> 3. **精排与置信度评估**。
2. **反思拦截**：若 `top_ce < 0.35`，说明改写模型未精准捕捉到财报或宏观数据的关键词。图进入**自旋锁分支**，让模型反思并自动重新翻译/改写 Query，返回步骤 2 重新检索（Max loop = 2）。



### 2.4 数学与策略计算工具外挂（Tool Calling 落地）

* 严禁 LLM 直接口算财务指标。
* 系统提供硬编码的 Python 工具函数（如：`kelly_criterion_calc` 凯利公式计算器、`grid_trading_cost_estimator` 网格成本计算器）。LLM 负责从 RAG 检索到的 Context 中提取数字，转换为 JSON 调用这些工具，再将精确的数学结果返回给用户。

## 4. 落地实施路线图：按部就班慢慢来

不要想着一天写完。既然你拥有强大的工程解耦思维，建议将此需求分为**四个敏捷迭代版本**推进：

### 🧱 阶段一：定义“骨架”状态（第 1 周）

* **目标**：不加任何新知识，先用 LangGraph 把你现有的 `ai_app1` 聊天流程框起来。
* **行动**：
1. 定义 `FinanceAgentState(TypedDict)`，把 `messages`, `rewritten_queries`, `retrieved_docs`, `loop_count` 放进去。
2. 写一个最简单的图：`Input -> 改写节点 -> 检索节点 -> 生成节点 -> Output`。
3. 验证你 v3.3 的 `retrieval_trace.py`（全链路 Trace）和 `latency_breakdown.py`（PhaseTimer）在 Graph 模式下依然能正常打印耗时。



### 📊 阶段二：录入数据与跨领域融合（第 2-3 周）

* **目标**：让资料库丰富起来，打通中英文混合路由。
* **行动**：
1. 去下载 2-3 份高清的科技股财报 Markdown/PDF，以及美联储的议息会议纪要（English）。
2. 用你的 `VectorIndexer` 统一灌入 `knowledge_base`，分别标记 `domain`。
3. 在“改写节点”中，调教你的本地 Qwen1.5B 拆分 Query。测试输入：“分析 NVIDIA 财报表现对 VIX 指数的影响”，看它能否并发出两路检索请求。



### 🔄 阶段三：实现自旋锁反思机制（第 4-5 周）

* **目标**：不浪费任何线上坏 Case，卡死 RAG 的底层质量。
* **行动**：
1. 在 LangGraph 中实现 `evaluate_and_rerank` 节点。
2. 提取你现有的 `FallbackReranker` 算出来的最高分 `top_ce`。如果用户问了一个很偏僻的财务指标导致分数只有 `0.21`，触发条件边（Conditional Edge），流转到 `query_reflection` 节点。
3. 让 Qwen 重新组织语言（比如把中文翻译成更地道的华尔街金融术语），然后连线回退，重新触发检索。



### 🧮 阶段四：挂载计算工具箱（第 6 周+）

* **目标**：让 Agent 具备真正的行动力，完成商业级闭环。
* **行动**：
1. 写两个纯 Python 函数：一个算凯利公式仓位，一个算网格头寸和做 T 后的基准成本。
2. 将其转化为大模型可感知的 Tool。
3. 最终测试复合大题：“根据 NVIDIA 最新营收指引带来的波动性，结合凯利公式，帮我算一下当前网格该如何分仓？”。观察主模型（MiniMax）是否能完美提取数字、调用工具、并给出严谨、无幻觉的流式金融报告。
